\section{Experiments}

\paragraph{Runs.}
We report three bounded API experiments plus a selected-message cross-model audit. The mixed pilot contains 50 scenes from the four initial scenario families. The perspective-stress replication contains 50 fresh perspective-shift scenes. The partial-observability support run contains 50 private-landmark scenes. In all runs, the speaker model generates candidates, local selectors choose final messages, and held-out API listeners evaluate those messages. The cross-model audit reuses the same speaker candidates and selected messages, then evaluates them with \texttt{gpt-4.1-nano}, \texttt{gpt-5.4-nano}, and \texttt{gpt-5.5} held-out listener prompts. We use paired bootstrap intervals over scenes for method differences.

\paragraph{Model versions and reproducibility.}
The cache records exact response model versions: \texttt{gpt-5.4-nano-2026-03-17} for the main API runs, \texttt{gpt-4.1-nano-2025-04-14} for the alternate-model audits, and \texttt{gpt-5.5-2026-04-23} for the frontier-listener follow-up. The repository includes generated prompt and schema documentation, a token-accounting report over 4,982 cached responses, an integrity audit that checks generated messages and listener choices, and an artifact guide mapping paper claims to source scenes, cached result files, tables, and verifier outputs. As a benchmark-scale artifact sanity check, the release also includes a no-API 600-scene balanced local sweep: mirror self-play has 1.000 same-play but 0.631 cross-play success, while population-play, template, and oracle candidate each reach 1.000.

\paragraph{What is being tested?}
The experiments do not test whether parameter updates on a language model improve pragmatic reasoning. Instead, they test a smaller but common mechanism: using interaction-derived success to select among possible messages. This mechanism appears in reranking, rejection sampling, self-refinement, and data filtering. If selection against one partner can inflate apparent success in a controlled reference game, then cross-play is a useful diagnostic before moving to more expensive training or richer social environments.

\section{Results}

\paragraph{Finding 1: same-play success overstates held-out communicative success.}
Table~\ref{tab:mixed} shows that mirror self-play obtains 1.000 same-play success but 0.907 cross-play success in the mixed pilot. Population-play obtains 1.000 cross-play success on the same scenes. The paired population-minus-mirror difference is 0.093 with a 95\% bootstrap interval of [0.027, 0.173].

Scenario-level results localize the effect. Unique-attribute and distractor-contrast scenes are at ceiling, relational-reference scenes show a smaller mirror gap, and perspective-shift scenes are the hard case: direct messages and shortest messages often under-specify the target, and mirror self-play selects messages that work for its fixed partner but fail under held-out listeners. This rules out a uniform degradation story; the self-play gap appears where listener perspective and ambiguity matter.

\paragraph{Finding 2: the gap replicates under fresh perspective-shift scenes.}
Table~\ref{tab:stress} confirms the pattern in 50 new perspective-shift scenes. Mirror self-play again achieves 1.000 same-play success but only 0.813 cross-play success. Population-play reaches 1.000 cross-play success, matching the oracle candidate upper bound. The paired population-minus-mirror difference is 0.187 with a 95\% bootstrap interval of [0.100, 0.287]. The direct baseline is also substantially weaker than population-play, showing that the gain is not simply due to the API speaker already producing a robust first candidate.

\paragraph{Finding 3: the effect persists under an alternate held-out model family.}
Table~\ref{tab:altmodel} evaluates the same perspective-stress candidate sets with an alternate held-out model, \texttt{gpt-4.1-nano}. Direct and shortest baselines become weaker, mirror self-play drops to 0.713 cross-play success, and population-play again reaches 1.000. The paired population-minus-mirror difference under this alternate held-out model is 0.287 with a 95\% bootstrap interval of [0.193, 0.380]. This audit is not meant to compare model quality; rather, it checks whether the observed partner-overfitting gap survives a held-out listener shift.

\paragraph{Finding 4: coordinate fallbacks are useful, but selector quality matters without them.}
The strongest population-play result uses candidate sets that include an explicit row/column fallback. Table~\ref{tab:nocoord} asks whether the result is merely a coordinate fallback effect. We remove candidate messages that explicitly mention row or column numbers and recompute selection without new API calls. This ablation is deliberately stricter and exposes a limitation of the current local population selector.

Robust non-coordinate candidates remain available: the no-coordinate oracle reaches 0.793, and consensus+info reaches 0.760. However, naive population-play falls to 0.420 because the local training listeners often assign equal or low scores to more informative non-coordinate descriptions, causing the length penalty to favor short ambiguous messages. Thus the headline population-play result should be read as robust fallback selection when explicit listener-invariant coordinates are available, while the ablation shows that cross-play is also useful for diagnosing selector weaknesses. The same cache-only filter on the mixed pilot gives a no-coordinate oracle of 0.960 and consensus+info success of 0.920, with gains over mirror concentrated in relational-reference and perspective-shift scenes.

Selection behavior explains the ablation. A candidate-role audit shows that, in the full perspective-stress and partial-observability runs, population-play selects the coordinate-fallback slot in 1.000 of scenes; in the alternate-model perspective audit, this is 50 coordinate selections versus 17 for mirror self-play. Once exact coordinates are removed, naive population-play selects a spatially informative message in only 4 of 50 perspective-stress scenes and drops to 0.420 cross-play success. Consensus+info falls back to the informativeness prior in all 50 stress scenes, selecting spatially informative messages throughout and reaching 0.760. In the mixed no-coordinate pilot, consensus+info uses local consensus in 32 scenes and the informativeness fallback in 18 scenes.

\paragraph{Failure decomposition: generation versus selection.}
The oracle result is important. In the perspective-stress replication, the oracle candidate succeeds on every scene, so the API speaker usually generated a robust message. The failure is selection: mirror self-play often selects a shorter message that is only partner-robust, whereas population-play selects the explicit fallback or a less frame-sensitive description.

A cache-only selection-regret audit makes this decomposition explicit. Regret is oracle candidate success minus method success, averaged by scene. In the full-candidate API runs, population-play has zero regret in the mixed, perspective-stress, alternate-model, and partial-observability settings, while mirror self-play leaves regrets of 0.093, 0.187, 0.287, and 0.333 respectively. In the no-coordinate ablations, consensus+info reduces regret to 0.040 in the mixed pilot, 0.033 in the perspective-stress audit, and 0.013 in partial observability, showing that most remaining failures are selector failures rather than missing candidates.

An even stricter candidate-pool audit asks whether any candidate succeeds with every held-out listener in a scene. Every full-candidate run has at least one such robust candidate in all 50 scenes, and population-play selects a robust candidate in 1.000 of scenes. Mirror robust-selection rates are 0.860, 0.720, 0.560, and 0.500. After exact coordinates are removed, robust non-coordinate candidates remain available in 0.920, 0.660, and 1.000 of scenes for the mixed, alternate-model perspective, and partial-observability runs, and consensus+info matches the per-scene oracle in 0.940, 0.920, and 0.980 of scenes.

\paragraph{Ruling out simple alternative explanations.}
A uniform-random candidate baseline rules out the explanation that any candidate would do. Random expected success is 0.865, 0.715, 0.657, and 0.753 in the four full-candidate runs, so population-play improves over random by 0.135, 0.285, 0.343, and 0.247. In the no-coordinate runs, consensus+info improves over random by 0.100, 0.218, and 0.316.

A message-length audit rules out a pure verbosity explanation. Population-play does select longer messages than mirror self-play by 1.26, 2.82, 2.82, and 3.16 whitespace tokens on average in the four full-candidate runs, but direct messages are longer than mirror in the perspective and partial-observability runs while remaining less successful. Population-play messages also remain near the deterministic template scale: 6.82, 10.22, 10.22, and 9.66 tokens on average, compared with a 10-token template baseline.

A prefix-$K$ candidate-budget audit shows why the four-candidate protocol is useful. In the alternate-model perspective audit, robust-scene coverage rises from 0.380 at $K=1$ to 0.660 at $K=3$ and 1.000 at $K=4$; in partial observability, it rises from 0.620 at $K=1$ to 1.000 at $K=3$. Candidate diversity matters, but only if the selector can identify the robust candidate.

A held-out listener-disagreement audit shows that mirror self-play produces less stable messages. Mirror self-play yields split held-out outcomes in 0.080, 0.180, 0.360, and 0.500 of scenes across the full-candidate runs, while population-play has 0.000 split outcomes in all four. In no-coordinate partial observability, consensus+info reduces split outcomes from mirror's 0.540 to 0.020. A listener-confidence audit further shows that self-reported uncertainty is not enough: in the alternate-model perspective runs, ambiguity flags are 0.000 while mirror self-play still produces high-confidence failures (confidence $\ge 0.8$) in 0.147 of full-candidate listener rows and 0.280 of no-coordinate listener rows.

\paragraph{API-listener leave-one-out mechanism check.}
To check that the result is not only an artifact of deterministic local selectors, we also run a cache-only leave-one-listener-out analysis over the API all-candidate evaluations. For each scene, one API listener prompt is held out; API-mirror selection uses one different API listener prompt, while API-population selection uses the other two. Population selection improves over API-mirror selection in all four cached runs. The gains are small in the mixed and main perspective runs, but larger for the alternate-model perspective audit (0.940 vs. 0.847) and partial observability (0.940 vs. 0.767). Because this analysis reuses the cached API listener pool for post-hoc splits, we treat it as a mechanism check rather than as the main held-out evaluation.

\paragraph{Partial-observability support check.}
As an additional support check, we run a 50-scene partial-observability stress setting in which the speaker sees a private landmark hidden from the listener. The prompt instructs the speaker to use only listener-visible information. In this setting, mirror self-play obtains 0.667 cross-play success despite 1.000 same-play success, while population-play and the oracle both reach 1.000. After removing exact row/column candidates, robust non-coordinate messages remain available: the no-coordinate oracle is 1.000 and consensus+info reaches 0.987. The candidate audit finds zero explicit private-landmark references, so the failures reflect under-informative messages that fit both the target and distractor rather than object-ID leakage or hidden-landmark leakage. Rubric coding supports this interpretation: all 50 full-run mirror failures and all 59 no-coordinate mirror failures are underspecified-distractor choices.
