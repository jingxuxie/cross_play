\large Cross-Play Evaluation of Communicative Self-Play in Situated Reference Games}

\begin{abstract}
Interactive language agents are often evaluated with the same simulated partners
that provide feedback or selection signals. In communicative settings, this can
conflate robust pragmatic competence with partner-specific conventions. We study
this issue in PRAG-CrossPlay, a controlled situated reference game where a
speaker must produce a natural-language message that lets a listener identify a
hidden target object. Because listener choices are scored against ground-truth
object IDs, the task gives an objective communicative-success signal without an
LLM judge. In a hybrid API experiment, an API speaker generates candidate
messages, a fixed simulated partner or a small listener population selects among
them, and held-out API listeners evaluate the selected messages. Mirror
self-play achieves perfect same-partner success but drops under cross-play,
especially on perspective-sensitive references. In a 50-scene perspective stress
replication, mirror self-play obtains 1.000 same-play success but only 0.813
cross-play success, while population-play reaches 1.000 cross-play success on
the same candidate sets; with a different held-out model family, mirror drops
further to 0.713 while population-play remains at 1.000. A no-coordinate
ablation shows that this result depends on the availability of explicit
listener-invariant fallbacks, and that selector quality matters when those
fallbacks are removed. These results suggest that cross-play should be a
standard diagnostic for language agents trained or selected through situated
interaction.
\end{abstract}

\section{Introduction}

Pragmatic communication is partner-sensitive: speakers choose utterances with
respect to what a listener can observe, infer, and disambiguate. Interactive
language-agent work increasingly uses simulated interaction to improve or
evaluate social behavior, including broad social environments such as
SOTOPIA~\cite{zhou2023sotopia} and interactive-learning variants such as
SOTOPIA-$\pi$~\cite{wang2024sotopiapi}. This creates a useful but risky feedback
loop. If the partner that supplies feedback has stable quirks, high interaction
success may reflect adaptation to that partner rather than robust pragmatic
competence.

A related warning appears in zero-shot coordination: self-play can induce
specialized conventions that do not transfer to independently trained or novel
partners, motivating cross-play and inter-algorithm cross-play evaluations
\cite{hu2020otherplay,lucas2022anyplay}. PRAG-CrossPlay brings this diagnostic
view to natural-language pragmatic communication. Instead of measuring team
reward in a game such as Hanabi, we measure whether a message lets held-out
listeners recover a ground-truth referent.

We test this issue in situated reference games, a classic setting for studying
pragmatic inference and informative utterance choice~\cite{frank2012predicting,
hu2021scalable}. A speaker sees a target object in a small grid scene and sends
a natural-language message to a listener, who must choose the intended object
from the listener-visible scene. The resulting success signal is objective:
the listener either chooses the target object ID or not. This lets us separate
same-partner success from cross-partner success without relying on an LLM judge.

Our core question is simple: when a speaker selects messages by simulating a
listener, does it learn robust pragmatics or just exploit that listener? We
compare a mirror self-play selector, which chooses the candidate that succeeds
with one fixed simulated listener, to a population-play selector, which chooses
the candidate that succeeds across a small heterogeneous listener population.
We then evaluate selected messages with held-out API listeners.

The paper contributes: (i) PRAG-CrossPlay, a released situated reference-game
benchmark with procedural scenes, prompts, cached API traces, and verification
scripts; (ii) a cross-play protocol that separates same-partner success from
held-out communicative success; and (iii) a compact empirical diagnosis showing
when self-play selection finds robust messages and when it instead selects
partner-specific under-informative descriptions.

The current experiments support three findings. First, same-play success can
overstate held-out communicative success: mirror self-play achieves perfect
same-partner success but loses under cross-play. Second, the gap is concentrated
in perspective-sensitive scenes, where short messages such as ``the small blue
sphere'' or speaker-frame relational descriptions work for the training partner
but fail for held-out listeners. Third, robust candidates are usually available,
but selector design matters: population-play closes the observed gap when exact
listener-invariant fallbacks are present, while a no-coordinate ablation shows
that a consensus-plus-informativeness selector is needed when those fallbacks
are removed.

\section{Related Work}

\paragraph{Interactive social-agent evaluation.}
SOTOPIA evaluates language agents in broad social role-play settings with goals
such as coordination, exchange, and competition~\cite{zhou2023sotopia}.
SOTOPIA-$\pi$ extends this line to interactive learning and reports that
LLM-based evaluators can overestimate agents trained through social
interaction~\cite{wang2024sotopiapi}. PRAG-CrossPlay is narrower by design: it
does not try to cover open-ended social intelligence, but gives an objective
ground-truth success signal for one core communicative problem.

\paragraph{Cross-play and conventions.}
Self-play can produce conventions that work with a training partner but fail
with independently trained or differently prompted partners. This concern is
central in zero-shot coordination and motivates cross-play style evaluations
such as Other-Play and Any-Play~\cite{hu2020otherplay,lucas2022anyplay},
building on a broader view of conventions in communication~\cite{lewis1969convention}.
Our contribution is to bring this diagnostic to natural-language reference
games with API language-model listeners.

\paragraph{Reference games and pragmatics.}
Reference games are a standard testbed for pragmatic language use because the
speaker must choose an utterance relative to the listener's alternatives and
context~\cite{frank2012predicting,goodman2016pragmatic}. Recent scalable pragmatic
communication work studies self-supervised utterance choice in reference-like
settings~\cite{hu2021scalable}. PRAG-CrossPlay complements these lines by
separating same-partner success from held-out listener success, making partner
overfitting directly measurable.

\section{PRAG-CrossPlay Task}

Each PRAG-CrossPlay episode contains a grid scene, visible objects, a target
object ID hidden from the listener, a speaker message, and a listener choice.
Objects have color, shape, size, row, and column attributes. The listener sees
the candidate objects but not the target label. The speaker is instructed not to
mention the hidden ID and to use only listener-visible information.

We generate five scenario families. Unique-attribute scenes have targets that
are identifiable by simple attributes. Distractor-contrast scenes include a
distractor sharing salient properties with the target. Relational-reference
scenes require descriptions relative to a landmark. Perspective-shift scenes
give the speaker and listener different orientations, making left/right and
other frame-sensitive descriptions brittle. Partial-observability scenes give
the speaker a private landmark that is hidden from the listener, testing whether
messages use only listener-visible information. The main stress tests in this
draft focus on perspective shift and partial observability because pilot
results showed ceiling effects in the easier families.

\paragraph{Evaluation.}
For each held-out listener response, success is
\[
  \mathbb{1}[\texttt{choice\_id} = \texttt{target\_id}].
\]
For self-play methods we also record same-play success under the listener used
for selection. The cross-play gap is
\[
  \text{same-play success} - \text{held-out cross-play success}.
\]
A positive gap indicates that same-partner success is overstating robustness.

\section{Methods}

\paragraph{API candidate generation.}
For each scene, an API speaker generates $K=4$ candidate utterances. The prompt
asks for a natural first attempt, an attribute-based message, a relational or
spatial message, and an explicit fallback. The prompt includes the target
attributes but not the target ID in candidate messages. All API calls are cached
by prompt hash.

\paragraph{Baselines.}
The template baseline describes the target with attributes and exact row/column.
The direct baseline uses the first API candidate. Best-of-$K$ shortest selects
the shortest generated candidate, controlling for candidate multiplicity without
listener feedback.

\paragraph{Informativeness prior and consensus+info.}
In the no-exact-coordinate ablations, we also report a text-only selector that
prefers candidates containing contrastive or spatial cues such as ``only,''
``leftmost,'' ``below,'' or ``next to.'' We then combine this prior with local
listener feedback: the consensus+info selector uses population feedback when a
candidate has unanimous local support, and otherwise falls back to the
informativeness prior. Neither selector uses held-out listener labels.

\paragraph{Mirror self-play.}
The mirror selector scores each candidate with a fixed simulated listener and
chooses the shortest candidate that the listener maps to the target. This is a
minimal model of same-partner communicative selection.

\paragraph{Population-play.}
The population selector scores each candidate across three heterogeneous local
listeners. These listeners differ in tie-breaking, frame assumptions, and
relation weighting. The selected message maximizes average success with a small
length penalty.

\paragraph{Oracle candidate.}
For analysis only, we evaluate every generated candidate with held-out API
listeners and report the best candidate's held-out success. This measures
whether failures come from candidate generation or candidate selection.

\section{Experiments}

We report three bounded API experiments. The mixed pilot contains 50 scenes
from the four initial scenario families. The perspective-stress replication
contains 50 new perspective-shift scenes. The partial-observability support run
contains 50 private-landmark scenes. In all runs, API speakers generate
candidates and held-out API listeners evaluate final messages. The selectors
are local and deterministic, which keeps cost low and makes the
partner-overfitting mechanism inspectable. We use paired bootstrap intervals
over scenes for method differences.

\paragraph{Reproducibility checks.}
All API responses are cached. The cache records exact response model versions:
\texttt{gpt-5.4-nano-2026-03-17} for the main API runs and
\texttt{gpt-4.1-nano-2025-04-14} for the alternate-model audit. We also include
a generated protocol appendix with prompts and JSONL schemas, plus a benchmark
integrity audit that checks generated messages for object-ID leakage, verifies
listener choices and success flags against source scenes, and confirms balanced
scene coverage across methods. A reviewer-facing artifact guide maps each main
claim to the concrete source scenes, cached result files, tables, and verifier
outputs used to support it. As a benchmark-scale artifact sanity check, the
release also includes a no-API 600-scene balanced local sweep: mirror self-play
has 1.000 same-play but 0.631 cross-play success, while population-play,
template, and oracle candidate each reach 1.000.

Table~\ref{tab:nocoord} tests whether this result is merely a coordinate
fallback effect. We remove one exact row/column candidate from each
perspective-stress candidate set and recompute selection without new API calls.
This ablation is deliberately stricter and exposes a limitation of the current
local selector. Robust non-coordinate candidates remain available: the
no-coordinate oracle reaches 0.793, and consensus+info reaches 0.760. However,
naive population-play falls to 0.420 because the local training listeners often
assign equal or low scores to the more informative non-coordinate descriptions,
causing the length penalty to favor short ambiguous messages. Thus the headline
population-play result should be read as robust fallback selection when explicit
listener-invariant coordinates are available, while the ablation shows that
selector quality matters when such fallbacks are removed. The same cache-only
filter on the mixed pilot gives a no-coordinate oracle of 0.960 and
consensus+info success of 0.920, with the gains over mirror concentrated in
relational-reference and perspective-shift scenes.

Selection behavior explains the ablation. A candidate-role audit makes this
concrete: in the full perspective-stress and partial-observability stress runs,
population-play selects the coordinate-fallback slot in 1.000 of scenes; in the
alternate-model perspective audit, this is 50 coordinate selections versus 17
for mirror self-play. Once exact coordinates are removed, naive population-play
selects a spatially informative message in only 4 of 50 perspective-stress
scenes and drops to 0.420 cross-play success. Consensus+info falls back to the
informativeness prior in all 50 stress scenes, selecting spatially informative
messages throughout and reaching 0.760; the same role audit shows it selects the
short attribute-only slot in 0.000 of the no-coordinate perspective and
partial-observability scenes. In the mixed no-coordinate pilot, consensus+info
uses local consensus in 32 scenes and the informativeness fallback in 18 scenes.

The oracle result is important. In the perspective-stress replication, the
oracle candidate succeeds on every scene, so the API speaker usually generated
a robust message. The failure is selection: mirror self-play often selects a
shorter message that is only partner-robust, whereas population-play selects
the explicit fallback or a less frame-sensitive description.

A cache-only selection-regret audit makes this decomposition explicit. Regret
is oracle candidate success minus method success, averaged by scene. In the
full-candidate API runs, population-play has zero regret in the mixed,
perspective-stress, alternate-model, and partial-observability settings, while
mirror self-play leaves regrets of 0.093, 0.187, 0.287, and 0.333 respectively.
In the no-coordinate ablations, consensus+info reduces regret to 0.040 in the
mixed pilot, 0.033 in the perspective-stress audit, and 0.013 in partial
observability, showing that most remaining failures are selector failures rather
than missing candidates.

An even stricter candidate-pool audit asks whether any candidate succeeds with
every held-out listener in a scene. Every full-candidate run has at least one
such robust candidate in all 50 scenes, and population-play selects a robust
candidate in 1.000 of scenes; mirror robust-selection rates are 0.860, 0.720,
0.560, and 0.500. After exact coordinates are removed, robust non-coordinate
candidates remain available in 0.920, 0.660, and 1.000 of scenes for the mixed,
alternate-model perspective, and partial-observability runs, and consensus+info
matches the per-scene oracle in 0.940, 0.920, and 0.980 of scenes.
A uniform-random candidate baseline rules out the explanation that any
candidate would do: random expected success is 0.865, 0.715, 0.657, and 0.753
in the four full-candidate runs, so population-play improves over random by
0.135, 0.285, 0.343, and 0.247; in the no-coordinate runs, consensus+info
improves over random by 0.100, 0.218, and 0.316.
A message-length audit rules out a pure verbosity explanation. Population-play
does select longer messages than mirror self-play by 1.26, 2.82, 2.82, and
3.16 whitespace tokens on average in the four full-candidate runs, but direct
messages are longer than mirror in the perspective and partial-observability
runs while remaining less successful. Population-play messages also remain near
the deterministic template scale: 6.82, 10.22, 10.22, and 9.66 tokens on
average, compared with a 10-token template baseline.
A prefix-$K$ candidate-budget audit shows why the four-candidate protocol is
useful: in the alternate-model perspective audit, robust-scene coverage rises
from 0.380 at $K=1$ to 0.660 at $K=3$ and 1.000 at $K=4$; in partial
observability, it rises from 0.620 at $K=1$ to 1.000 at $K=3$.
A held-out listener-disagreement audit shows the same pattern at the selected
message level. Mirror self-play produces split held-out outcomes in 0.080,
0.180, 0.360, and 0.500 of scenes across the full-candidate runs, while
population-play has 0.000 split outcomes in all four. In no-coordinate partial
observability, consensus+info reduces split outcomes from mirror's 0.540 to
0.020.
A listener-confidence audit further shows that self-reported uncertainty is not
enough: in the alternate-model perspective runs, ambiguity flags are 0.000 while
mirror self-play still produces high-confidence failures (confidence $\ge 0.8$)
in 0.147 of full-candidate listener rows and 0.280 of no-coordinate listener
rows.

To check that this is not only an artifact of the deterministic local selector,
we also run a cache-only leave-one-listener-out analysis over the API
all-candidate evaluations. For each scene, one API listener prompt is held out;
API-mirror selection uses one different API listener prompt, while
API-population selection uses the other two. Population selection improves over
API-mirror selection in all four cached runs. The gains are small in the mixed
and main perspective runs, but larger for the alternate-model perspective audit
(0.940 vs. 0.847) and partial observability (0.940 vs. 0.767). Because this
analysis reuses the cached API listener pool for post-hoc splits, we treat it
as a mechanism check rather than as the main held-out evaluation.

As an additional support check, we also run a 50-scene partial-observability
stress check in which the speaker sees a private landmark hidden from the
listener. The prompt instructs the speaker to use only listener-visible
information. In this setting, mirror self-play obtains 0.667 cross-play success
despite 1.000 same-play success, while population-play and the oracle both
reach 1.000. After removing exact row/column candidates, robust non-coordinate
messages remain available: the no-coordinate oracle is 1.000 and
consensus+info reaches 0.987. The candidate audit finds zero explicit
private-landmark references, so the observed failures again reflect
under-informative messages that fit both the target and distractor rather than
object-ID leakage or hidden-landmark leakage. Rubric coding supports this
interpretation: all 50 full-run mirror failures and all 59 no-coordinate mirror
failures are underspecified-distractor choices.

\section{Qualitative Analysis}

In scene \texttt{ps\_000005}, the target is a small green sphere at row 4,
column 1, with another small green sphere at row 1, column 4. Mirror self-play
selects ``It's a small green sphere.'' Its local partner resolves this
correctly, but two of the three alternate-model held-out listeners choose the
distractor. Population-play selects ``The target is the small green sphere at
row 4, column 1,'' which all held-out listeners map to the target.

In scene \texttt{ps\_000011}, mirror self-play selects ``Find the large yellow
cube left of the purple cylinder.'' Under perspective shift, this relation is
frame-sensitive: one alternate-model held-out listener chooses the same-shape
same-color distractor. Population-play selects ``Go to row 4, column 1: the
large yellow cube,'' which all held-out listeners map to the target.

\section{Limitations}

The current experiments are intentionally small and diagnostic. The selector
partners are local deterministic listeners rather than trained LLM agents. This
is useful for isolating partner overfitting, but it means the result should be
framed as a cross-play diagnostic rather than as evidence about large-scale
agent training. The held-out listeners are API language models, not humans. We
audit two held-out model families, but both are still API LLMs and prompt
variants rather than independent human partners. The strongest population-play
result relies on exact coordinate fallbacks; without those fallbacks, the current
local listener population is not sufficient, although consensus+info recovers
much of the no-coordinate oracle. The grid scenes are simplified and
text-rendered, so the benchmark tests situated reference rather than embodied
perception. Finally, the strongest evidence is concentrated in two diagnostic
families, perspective shift and partial observability; broader claims require
larger balanced runs across all scenario types.

\section{Conclusion}

Same-partner communicative success is not sufficient evidence of robust
pragmatic communication. In PRAG-CrossPlay, mirror self-play can achieve perfect
same-play success while failing under held-out listeners, especially when
messages are perspective-sensitive or under-specified. Population-play mitigates
this failure when listener-invariant fallbacks are available, but the
no-coordinate ablation shows that cross-play is also useful for diagnosing and
repairing weaknesses in the selector itself. These results support cross-play as
a simple, objective evaluation axis for language agents that learn or select
messages through situated interaction.
