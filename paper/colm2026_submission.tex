\large Cross-Play Evaluation of Communicative Self-Play in Situated Reference Games}

\begin{abstract}
Language agents are increasingly trained, filtered, or evaluated through interaction with simulated partners. In pragmatic communication, however, success with the training partner can reflect a private convention rather than a generally interpretable message. We study this issue in PRAG-CrossPlay, a situated reference-game benchmark where a speaker describes a hidden target object and a listener must choose the target from a text-rendered scene. Success is objective: the listener's object ID is checked against ground truth, without an LLM judge. We compare mirror self-play, which selects utterances using one fixed simulated listener, with population-play, which selects utterances using a heterogeneous listener population, and evaluate both with held-out model listeners. The main pattern is simple: mirror self-play reaches 1.00 same-partner success but loses under cross-play, while population-play remains robust. Across \texttt{gpt-4.1-nano}, \texttt{gpt-5.4-nano}, and \texttt{gpt-5.5} held-out listeners, mirror cross-play ranges from 0.65--0.81 on the hardest settings, whereas population-play reaches 1.00 in every row. Stronger \texttt{gpt-5.5} speaker generation nearly solves direct first-candidate reference (0.99) but still leaves a mirror-selection gap (0.85 versus 1.00). A no-coordinate stress test shows that robust non-coordinate messages are available, but selector design remains the bottleneck. These results support cross-play as a compact diagnostic for distinguishing robust pragmatic communication from partner-specific shortcuts.
\end{abstract}

\section{Introduction}

A language agent can be successful in an interaction for the wrong reason. Suppose a speaker says ``the small green sphere'' in a scene with two small green spheres. A particular partner may still choose the intended object because of a hidden tie-breaking rule or a shared history. The same utterance can then fail with another listener who sees the same alternatives but resolves ambiguity differently. This is a pragmatic failure, not a grammatical one: the utterance is fluent, but it is not robustly informative for the listener distribution.

This distinction matters for current language-agent research because interaction is often used as a source of feedback. A simulated user can answer questions, provide reward, select preferred responses, or stand in for a social partner. These feedback loops are attractive because they move evaluation beyond static question answering and toward situated, consequential language use. They also create a risk: if the partner that supplies the feedback has stable quirks, high interaction reward may measure adaptation to that partner rather than robust pragmatic skill.

The risk is familiar from both pragmatics and multi-agent learning. Pragmatic theories emphasize informative utterance choice relative to listener knowledge, common ground, and distractor alternatives~\cite{grice1975logic,clark1996using,dale1995computational,frank2012predicting,goodman2016pragmatic}. Human speakers also adapt to particular partners, forming conceptual pacts that can be useful within an interaction but partner-specific across interactions~\cite{clark1986referring,brennan1996conceptual,pickering2004toward}. In multi-agent learning, self-play can produce conventions that work internally but transfer poorly to novel partners, motivating cross-play and zero-shot coordination evaluations~\cite{foerster2016learning,lazaridou2017multiagent,lowe2019pitfalls,hu2020otherplay,lucas2022anyplay}. We bring this diagnostic to natural-language reference: when communicative success is used to select a message, does that success transfer to new listeners?

We study this question in PRAG-CrossPlay, a controlled situated reference-game benchmark. A speaker sees a text-rendered grid scene, a hidden target object ID, and the target's visible attributes. The listener sees the candidate-object list and the speaker message, but not the target ID. The task deliberately abstracts away perception so that the experiment isolates pragmatic reference. Because the listener returns an object ID, success is scored objectively by checking whether the chosen ID equals the target.

The key distinction is between \emph{same-play} and \emph{cross-play}. Same-play asks whether a message works with the simulated listener used for selection. Cross-play asks whether that same message works with held-out listener prompts and model families. We compare \emph{mirror self-play}, which keeps the shortest candidate that succeeds with one fixed local listener, to \emph{population-play}, which chooses candidates that succeed across three heterogeneous local listeners. This is not a large-scale training procedure. It is a minimal version of a common agent pattern: generate candidate actions, simulate a partner, and keep the action that appears to succeed.

Our contributions are:

\par\noindent\hangindent=1.2em\hangafter=1\textbullet\ \textbf{A diagnostic benchmark.} PRAG-CrossPlay provides procedural situated reference games, prompt templates, cached traces, and verification scripts.

\par\noindent\hangindent=1.2em\hangafter=1\textbullet\ \textbf{A cross-play protocol.} We separate same-partner success from held-out listener success, making partner overfitting directly measurable.

\par\noindent\hangindent=1.2em\hangafter=1\textbullet\ \textbf{An empirical diagnosis.} Mirror self-play can achieve perfect same-play success while selecting messages that fail under held-out listeners; population-play removes the gap when robust candidates are selected, and no-coordinate stress tests reveal when selector quality is the remaining bottleneck.

\section{Related Work}

\paragraph{Pragmatics, reference, and common ground.}
Pragmatic communication depends on what a speaker expects the listener to observe, infer, and take as relevant~\cite{grice1975logic,clark1996using}. Referring expression generation formalizes this problem by choosing attributes that distinguish a target from distractors~\cite{dale1995computational}. Rational Speech Act and neural listener--speaker models further cast pragmatic interpretation as recursive reasoning about alternative utterances and meanings~\cite{frank2012predicting,goodman2016pragmatic,andreas2016reasoning,monroe2017colors,cohngordon2018pragmatically}. PRAG-CrossPlay follows this reference-game tradition but shifts the evaluation question from ``does a listener understand?'' to ``does a message selected for one listener transfer to other listeners?''

\paragraph{Partner adaptation and convention overfitting.}
Human dialogue involves adaptation: speakers and listeners coordinate lexical choices, build common ground, and converge on partner-specific descriptions~\cite{clark1986referring,brennan1996conceptual,pickering2004toward}. Such adaptation is useful within a conversation, but it also motivates the central concern in this paper. A convention that works with one partner can be a liability with another. Emergent-communication work similarly shows that self-play can induce successful but difficult-to-interpret protocols~\cite{foerster2016learning,lazaridou2017multiagent,havrylov2017emergence,lowe2019pitfalls}. Zero-shot coordination work addresses this issue by evaluating transfer to unseen partners~\cite{carroll2019utility,hu2020otherplay,lucas2022anyplay}. We apply the same cross-play lens to natural-language pragmatic reference.

\paragraph{Interactive language agents and model-based feedback.}
Modern language-agent systems often use model-based interaction, self-reflection, or generate-and-filter loops to improve behavior~\cite{yao2023react,madaan2023selfrefine,shinn2023reflexion}. Social-agent benchmarks such as SOTOPIA and SOTOPIA-$\pi$ evaluate richer role-play and interactive learning, while LLM-as-judge work highlights both the usefulness and fragility of model-based evaluation~\cite{zhou2023sotopia,wang2024sotopiapi,zheng2023judging}. PRAG-CrossPlay is narrower: it does not claim to measure broad social intelligence. Its purpose is to give a small objective test of one communicative mechanism and to expose when apparent interaction success is partner-specific.

\section{PRAG-CrossPlay Task}

\paragraph{Episode structure.}
Each episode contains a scene $s$, a target object $t$, a speaker message $u$, and a listener choice $\hat{t}$. The scene contains objects $O=\{o_1,\ldots,o_n\}$, each with a color, shape, size, row, column, and object ID. The speaker receives the target ID and target attributes so it knows which object to describe. The listener receives the candidate-object list and the message, but not the target ID. The speaker is instructed not to mention object IDs and to use only listener-visible information.

The benchmark is text-rendered. Rows and columns are explicit, and some scenes include different speaker and listener orientations. This abstraction lets us isolate linguistic reference from visual perception. It also makes every episode auditable: scripts check message leakage, valid listener choices, success labels, and coverage across scenario families.

\paragraph{Scenario families.}
We generate five families. \emph{Unique-attribute} scenes are identifiable by simple attributes. \emph{Distractor-contrast} scenes include another object sharing salient properties with the target. \emph{Relational-reference} scenes require a description relative to a visible landmark. \emph{Perspective-shift} scenes give speaker and listener different orientations, making left/right and other frame-sensitive descriptions brittle. \emph{Partial-observability} scenes include a private landmark visible to the speaker but hidden from the listener. The hardest paper-facing evidence focuses on perspective shift and partial observability, because easier families quickly reach ceiling.

\paragraph{Metrics.}
For a listener $\ell$, a message succeeds when the listener chooses the target:
\[
  \operatorname{succ}(u,s,t,\ell)=\mathbb{1}[\ell(u,s)=t].
\]
For each method, cross-play success is the mean success over scenes and held-out listener prompts. For self-play methods, same-play success is measured with the training listener used for selection. The cross-play gap is
\[
  \operatorname{gap}=\text{same-play success}-\text{held-out cross-play success}.
\]
A positive gap means same-partner success overstates robustness. We also use oracle candidate success to separate generation failures from selection failures: if some generated candidate succeeds with held-out listeners but a method chooses a failing candidate, the bottleneck is selection.

\section{Methods}

\paragraph{Candidate generation.}
For each scene, an LLM speaker generates $K=4$ messages: a natural first attempt, a concise attribute description, a relational or spatial description, and an explicit fallback that may use row/column coordinates. Each message is limited to 25 words. The target ID appears in the prompt only so the speaker knows which object to describe; generated messages are audited for ID leakage.

\paragraph{Held-out listeners.}
Held-out listeners receive the listener-visible scene, orientations, and one speaker message. They return a JSON choice ID and auxiliary fields such as confidence or ambiguity flags. Different listener prompts use different tie-breaking and comparison styles. Importantly, held-out models are used as listeners, not judges: they choose an object, and the benchmark code scores the choice against ground truth.

\paragraph{Training listeners and selectors.}
The self-play selectors use deterministic local listeners. These listeners differ in tie-breaking, spatial-frame assumptions, and relation weighting. Using local listeners keeps the feedback signal cheap and inspectable.

\emph{Direct} uses the first generated message. \emph{Best-of-$K$ shortest} selects the shortest candidate, controlling for candidate multiplicity without listener feedback. \emph{Template} uses attributes plus exact row/column coordinates. \emph{Mirror self-play} selects the shortest candidate that succeeds with one fixed local listener. \emph{Population-play} selects the candidate with highest average success across three heterogeneous local listeners, with a small length penalty for ties. \emph{Consensus+info}, used in no-coordinate ablations, falls back to a text-only informativeness prior when local listener consensus is weak. \emph{Oracle candidate} selects the best candidate under held-out listeners and is reported only as an analysis upper bound.

\section{Experiments}

We report five compact experiments. The mixed pilot contains 50 scenes from the initial scenario families. The perspective-stress replication contains 50 fresh perspective-shift scenes. The partial-observability support run contains 50 private-landmark scenes. A cross-model audit reuses selected messages and evaluates them with \texttt{gpt-4.1-nano}, \texttt{gpt-5.4-nano}, and \texttt{gpt-5.5} held-out listeners. Finally, two \texttt{gpt-5.5} follow-ups test whether stronger speaker generation or no-coordinate candidate diversity changes the diagnosis.

All calls are cached. The cache records exact model versions: {\footnotesize\texttt{gpt-5.4-nano-2026-03-17}}, {\footnotesize\texttt{gpt-4.1-nano-2025-04-14}}, and {\footnotesize\texttt{gpt-5.5-2026-04-23}}. The repository includes prompt documentation, token accounting over 7,171 cached responses, integrity audits, and an artifact guide mapping claims to result files. The experiments test selection from communicative feedback, not parameter training: the question is whether a message selected because it works with one partner also works with held-out partners.

\section{Results}

The results are easiest to read as three takeaways.

\paragraph{Takeaway 1: mirror self-play looks perfect until the partner changes.}
In the mixed pilot, mirror self-play obtains 1.00 same-play success but only 0.91 cross-play success. The gap is concentrated in perspective-sensitive scenes; unique-attribute and distractor-contrast scenes are near ceiling. In 50 fresh perspective-shift scenes, mirror again has 1.00 same-play success but drops to 0.81 cross-play, while population-play reaches 1.00. This shows why same-play alone is misleading: the selected message can work for the training listener while remaining under-informative for other listeners.

\paragraph{Takeaway 2: the gap survives held-out model shift, including GPT-5.5.}
Table~\ref{tab:crossmodel} summarizes the main result. Across two hard settings and three held-out listener families, population-play reaches 1.00 cross-play success in every row. Mirror self-play ranges from 0.65 to 0.81 despite perfect same-play success. The GPT-5.5 listener rows are especially useful: they show that a stronger current listener does not simply repair the mirror-selected ambiguities. A failure-overlap audit supports the same conclusion: most earlier mirror-failure scenes also fail under GPT-5.5, and GPT-5.5 population-play has zero scene-level failures in both hard settings.

\paragraph{Takeaway 3: stronger generation helps, but selection remains the bottleneck.}
The GPT-5.5 speaker audit regenerates all 50 perspective-stress candidate sets with a stronger speaker. Direct first-candidate success rises to 0.99 and robust candidates appear by $K=2$ in every scene. Nevertheless, mirror self-play still reaches only 0.85 cross-play success despite 1.00 same-play success, while population-play reaches 1.00. Thus the problem is not simply that the speaker cannot produce good messages; the selector can still choose a partner-specific one.

\paragraph{No-coordinate stress test.}
The original population-play result often uses row/column fallbacks, which are listener-invariant in the text grid. To test whether this is doing all the work, we run a stricter GPT-5.5 no-coordinate audit. A dedicated K=8 prompt forbids exact row/column language and generates 400 candidates across 50 perspective-stress scenes with zero coordinate violations. The oracle remains 1.00, so robust non-coordinate descriptions are available. Table~\ref{tab:nocoord_summary} shows that increasing no-coordinate diversity improves mirror and population selection, but also reveals that heuristics are not monotonic: consensus+info is strong at K=4 but worse at K=8 because it can over-select brittle long relational descriptions. The safe conclusion is that cross-play diagnoses whether the bottleneck is generation or selection.

\paragraph{Audits rule out simpler explanations.}
The oracle, candidate-pool, and regret audits point to selection rather than missing candidates. In full-candidate runs, population-play has zero regret against the oracle, whereas mirror leaves substantial regret on hard settings. Random-candidate baselines show that not every candidate works; message-length audits show that the gain is not just verbosity; and listener-disagreement audits show that mirror-selected messages often split held-out listeners while population-selected messages do not. A leave-one-listener-out analysis over cached API listener evaluations also favors population selection over mirror selection. These audits are included in the repository to keep the main paper readable while preserving the evidence trail.

\section{Qualitative Analysis}

The failure mode is visible in individual scenes. In scene \texttt{ps\_000005}, the target and a distractor are both small green spheres. Mirror self-play selects ``It's a small green sphere.'' Its local partner resolves this correctly, but held-out listeners choose the distractor. Population-play selects ``The target is the small green sphere at row 4, column 1,'' which all held-out listeners map to the target. The mirror message is syntactically valid but pragmatically under-informative.

In scene \texttt{ps\_000011}, mirror self-play selects ``Find the large yellow cube left of the purple cylinder.'' Under perspective shift, this relation is frame-sensitive: a held-out listener can interpret left/right in a different frame and choose the distractor. Population-play selects a coordinate-grounded description that all held-out listeners resolve correctly.

These examples show two ways same-partner success can mislead. The training listener may resolve ambiguity with a target-favoring tie-breaker, or it may share the speaker's spatial frame. Both produce perfect same-play success without robust communicative success. The coded failures are concentrated: among 152 listener-level mirror failures, 147 are underspecified distractor cases and 5 are perspective-frame errors. A rule-based ambiguity verifier using visible attributes, coordinate mentions, and left/right cues recovers all coded mirror-failure labels, giving a non-LLM check on the taxonomy.

\section{Discussion}

PRAG-CrossPlay supports a simple evaluation principle: communicative success should specify the partner distribution. Mirror self-play often has a robust message available in the candidate pool, but selects a shorter or frame-sensitive message that works for one partner. Population-play improves robustness by requiring success across multiple listener assumptions. The no-coordinate experiments sharpen the point: when explicit listener-invariant coordinates are removed, good descriptions can still be generated, but selector quality determines whether they are chosen.

This matters beyond reference games. Many language-agent pipelines generate alternatives, simulate a partner or critic, and keep the response that appears to work. If the same partner type is used for evaluation, the pipeline can reward private conventions, hidden tie-breakers, or brittle perspective assumptions. Cross-play is a low-cost diagnostic: hold out partner prompts, model families, or humans, and report the same-play/cross-play gap.

\section{Limitations}

The experiments are intentionally small and diagnostic. The selector partners are deterministic local listeners rather than trained LLM agents, so the result should be framed as a cross-play diagnostic rather than as evidence about large-scale agent training. The held-out listeners are language models, not humans. We audit three held-out model families including \texttt{gpt-5.5}, but they are still model listeners and prompt variants rather than independent human partners. The strongest ceiling results use exact coordinate fallbacks, and the K=8 no-coordinate audit is limited to 50 perspective-stress scenes. The grid scenes are text-rendered, so the benchmark tests situated reference rather than embodied perception. Broader claims require larger balanced runs across scenario types and human listener validation.

\section{Conclusion}

Same-partner communicative success is not sufficient evidence of robust pragmatic communication. In PRAG-CrossPlay, mirror self-play can achieve perfect same-play success while failing under held-out listeners, especially when messages are perspective-sensitive or under-specified. Population-play mitigates this when robust listener-visible candidates are selected, and no-coordinate stress tests show that cross-play also diagnoses selector weaknesses. Cross-play should therefore be a standard evaluation axis for language agents that learn, filter, or select messages through situated interaction.
